Accessibility Guidelines

Our design system follows WCAG guidelines for accessibility compliance.

Key Accessibility Features

  • All color combinations meet WCAG AA contrast requirements (4.5:1 minimum)
  • Focus indicators are clearly visible and consistent
  • Interactive elements have minimum touch target size of 44px
  • Motion respects user preferences with reduced motion support
  • High contrast mode support for users who need it

Usage Guidelines

Best practices for implementing the design system consistently.

Do's and Don'ts

✅ Do

  • Use design tokens (CSS variables) for colors and spacing
  • Follow the component class naming conventions
  • Test components in both light and dark themes
  • Ensure proper focus management for accessibility
  • Use semantic HTML elements where appropriate

❌ Don't

  • Hard-code colors or spacing values
  • Create custom button styles without good reason
  • Skip accessibility testing
  • Mix different design patterns inconsistently
  • Override system styles with !important unless necessary

Implementation

How to integrate and use the design system in your projects.

Integration Steps

  1. Include the design system CSS file in your HTML head:
    <link rel="stylesheet" href="anwalts-design-system.css">
  2. Use standardized class names for components:
    <button class="btn btn-primary btn-lg">Action</button> <div class="card"> <div class="card-header"> <h3 class="card-title">Title</h3> </div> <div class="card-body">Content</div> </div>
  3. Follow consistent patterns for new components
  4. Test across different screen sizes and accessibility tools
